# Week 2 (W22): May 27–30, 2026 — Infrastructure Sprawl

A week that starts with secrets pipelines and ends with Core Web Vitals: four days of compounding scope, five major arcs running simultaneously, and a codebase that gets measurably faster and considerably more complex at the same time.

---

## Narrative

### The Knowledge Graph Arrives

The week opens with a gesture toward orientation. Before anything else gets built, Claude runs `graphify .` on the full codebase — 150 files, 597 AST nodes, 1,126 edges, 96 detected communities. The run takes about 20 minutes and produces a knowledge graph in `graphify-out/`. From this point forward, codebase questions go through the graph first. It's less an event than a commitment: the project is now large enough that reading files cold is inefficient, and everyone apparently agrees to that assessment without much discussion.

### The CI/CD Marathon

The Infisical integration that ended Week 1 with five open code review findings immediately escalates. Felix has a clear agenda: GitHub Actions should fetch secrets via OIDC instead of storing them as GitHub Secrets. What follows is a long sequence of short sessions — S88 through S121, sixteen sessions in under two hours — that reads less like implementation and more like debugging a configuration that doesn't behave as documented. OIDC subject claim scoping, discovery URL mismatches, hardcoded environment slugs, permissions missing from workflow files. Obs 430 fixes the hardcoded `env-slug`; obs 434–435 add the missing `id-token: write` permissions; obs 436 documents that OIDC wasn't actually configured on the Infisical identity at all. The sessions end not with a working pipeline but with a PR review of the PWA feature — Felix pivots mid-debug rather than continuing to push on a misconfigured identity. The OIDC integration remains in an uncertain state as of May 27.

May 28 formalizes the PWA work. Two independent PR reviews happen in the same evening: one for `feature/offline-cache-options` (the useCacheStorage composable and PwaCacheOverview) and one immediately after for a second PR (`improvement/pwa-overview-more-data`) that adds richer cache analytics. The first review generates obs 440–460: type design issues, missing discriminated unions, an icon loading convention violation, and a sequential failure vulnerability in the CI workflow. Most are fixed in the same session. The `@nanostores/async` migration also happens here: obs 462–484, midnight on May 29, migrating four deprecated `computed() + task()` calls to `computedAsync`. The migration is clean — 145 tests pass — and committed as `f20bc97`.

### The Accordion Saga

The single most elaborate arc of the week is the accordion component, and it spans most of May 29 afternoon. It begins with a practical problem: the PWA cache bucket list needs expand/collapse behavior. The first fix is a click handler. Then an animation. Then the animation gets refactored from JavaScript to CSS. Then CSS to a named Vue transition. Then back to JavaScript. Then the existing `AccordionItem.vue` is discovered — it already exists, it references a missing SCSS file, and it doesn't use VueUse composition or CSS Grid.

Rather than patching the existing component, the decision (obs 572) is to build a proper system: `BaseAccordion`, `AccordionItem`, `AccordionHeader`, `AccordionTrigger`, `AccordionContent` — five components, a dedicated `src/components/accordion/` directory, full WAI-ARIA compliance, `useId()` for unique ID generation, CSS Grid animations, an SCSS file that actually exists this time, and a documentation file. Tests written. Old files deleted. `PwaCacheOverview` migrated to the new system. Total elapsed time: approximately two hours.

The accordion work exposes something about Felix's feedback style this week. In Week 1, sessions tended to run until a thread was exhausted. In Week 2, Felix issues short corrective prompts mid-implementation — "accessibility terminology," "single vs multiple open items," "styles import from global to local scope" — that redirect without restarting. The sessions are shorter and more surgical on the human side.

### emdash: An Experiment at the Edges

Late May 29 sees the strangest arc of the week: an attempt to integrate emdash CMS with a Node.js standalone adapter, replacing the Cloudflare Pages deployment target. The pivot is fast and ambitious — obs 779–875 represent roughly an hour of work that installs `better-sqlite3`, switches the Astro adapter to Node.js, configures emdash in `astro.config.mjs`, creates a `src/live.config.ts`, and tries to start a server.

The obstacles are instructive. The `@astrojs/react` integration conflicts with Vue during prerendering. `unplugin-icons` components break under SSR because they access browser APIs during static rendering. The fix — replacing icon imports with inline SVGs across four Astro templates — works but is mechanical and leaves residue. By obs 874–875, the build succeeds and the server runs, but with the AstroPWA plugin disabled. emdash is technically integrated, but the tradeoffs are visible: Node.js means no Cloudflare Pages, no PWA, and a React dependency injected into a Vue project. Nothing is committed to main.

### Performance: The Final Push

May 30 pivots hard to performance. A Lighthouse mobile audit surfaces failing Core Web Vitals: CLS, LCP, and network dependency issues. What follows is six hours of diagnosis and optimization, loosely organized but directionally consistent.

The first confirmed fix is structural: a wildcard store import in `_app.ts` was eagerly bundling all Nanostores into the main chunk. Obs 942–943 make the devtools and stores lazy-loaded. Obs 950 adds manual code splitting for the word-list search bundle. Obs 953 eliminates the `@stores/index` barrel entirely — 16 components refactored to import directly from `@stores/wordList.ts`. Three test files break and are fixed (obs 958). The barrel import convention in CLAUDE.md gets updated to match the new reality (obs 932).

The search index also has a race condition: concurrent async computations can fire duplicate fetches for `api/search/index.json`. Obs 963 patches the waterfall fetch pattern. `word-of-the-day` auto-fetches on mount without deferral (obs 922) — noted but the fix isn't logged this week.

By the end of May 30, the site has been deployed to Cloudflare Pages (obs 937–938), the bundle is split, the barrel is gone, and the linter has been satisfied. But the Lighthouse audit is still running at the close of observations. CLS and LCP improvements are in progress; whether they cross the thresholds isn't confirmed.
