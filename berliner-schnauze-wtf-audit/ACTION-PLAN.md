## Phase 1 — Critical Fixes · Week 1

### P1.1 Fix BreadcrumbList schema [30 min · CRITICAL]
**Impact:** Unblocks breadcrumb rich results for all 916 word pages.

Two bugs combined in the BreadcrumbList template generated by `astro-breadcrumbs`:
- Last item `@id` has `.html` suffix (e.g. `/wort/aalen.html` instead of `/wort/aalen`)
- All `@id` values are relative paths instead of required absolute IRIs

**Fix:** Replace `astro-breadcrumbs` JSON-LD output with a manually constructed schema block in `[...wordSlug].astro` and `wort/index.astro` using `canonicalUrl()` for every `item.@id`.

```ts
// Example fix in [...wordSlug].astro
const breadcrumbSchema = {
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "item": { "@id": "https://berliner-schnauze.wtf/", "name": "Start" } },
    { "@type": "ListItem", "position": 2, "item": { "@id": "https://berliner-schnauze.wtf/wort", "name": "Wort" } },
    { "@type": "ListItem", "position": 3, "item": { "@id": canonicalUrl(`/wort/${word.slug}`), "name": word.name } }
  ]
};
```

### P1.2 Allow AI search bots in robots.txt [5 min · CRITICAL]
**Impact:** Unlocks ChatGPT Search, Perplexity, Google AI Overviews visibility.

The current robots.txt blocks both training crawlers (correct) and real-time search crawlers (accidental). `GPTBot` serves both ChatGPT training and ChatGPT Search — blocking it removes the site from ChatGPT search answers.

**Add to `public/robots.txt`:**
```
User-agent: OAI-SearchBot
Allow: /

User-agent: ChatGPT-User
Allow: /

User-agent: PerplexityBot
Allow: /
```

Retain `GPTBot Disallow` and `ClaudeBot Disallow` if training opt-out is desired. The TDMREP `Content-Signal: search=yes, ai-train=no` header continues to express intent.
